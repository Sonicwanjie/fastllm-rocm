# 2026-05-25 HIP Compilation Full Restoration

## Result: 15/15 HIP files compiled, hip_fastllm.lib (10.6 MB) created

## Problem
All 29 HIP patches from 2026-05-22 were lost (never `git commit`-ed). Working directory reverted to original hipify output. 15/15 HIP files failed to compile.

## Solution: Two-Phase Restoration

### Phase 1: Core Header & Script Fixes

**`include/devices/cuda/fastllm-hip.h`** - Complete rewrite:
- `__nv_bfloat16` struct: `__host__ __device__` constructors, copy/move assignment, `float`/`hip_bfloat16` conversion
- `__nv_bfloat162` struct: `.x`/`.y` union fields, constructors from `(uint16_t, uint16_t)` and `(__nv_bfloat16, __nv_bfloat16)`
- FP8 type aliases: `__nv_fp8_e4m3` -> `__hip_fp8_e4m3`, `__nv_fp8_e5m2` -> `__hip_fp8_e5m2`
- All union types with default constructors: `union_char4/8`, `union_half4/8`, `union_bf16_4/8`, `union_bf16_4_fp16`
- Inline shuffle functions with `width=32` default parameter (3-arg call compat)
- `hipblasGemmEx` wrapper accepting `hipblasComputeType_t` (not `hipDataType`)
- `hipblasHgemm`/`hipblasHgemmStridedBatched` wrappers with `hipblasHalf` conversion
- `#define` macros: `checkCudaErrors`, `HIPBLAS_GEMM_DEFAULT_TENSOR_OP`, `__grid_constant__`, `cudaErrorNotSupported`

**`include/devices/cuda/fastllm-cuda.cuh`** - USE_ROCM branch:
- `#elif defined(USE_ROCM)` includes `fastllm-hip.h`, typedefs `cublasHandle_t`
- `showError(hipError_t, ...)` declaration
- Kernel extern declarations visible under `USE_ROCM`
- `CUDA_MAX` macro

**`include/devices/cpu/alivethreadpool.h`** - Added `#include <chrono>`

### Phase 2: ROCm FlashInfer Integration (Key Breakthrough)

Replaced `third_party/flashinfer` with headers from [ROCm/flashinfer](https://github.com/ROCm/flashinfer) which provides:
- `gpu_iface/` abstraction layer: portable CUDA/HIP math, utils, vec_dtypes
- `sampling.cuh` using `hipcub` instead of NVIDIA `cub`
- `PLATFORM_HIP_DEVICE` macro for HIP-specific code paths
- `allocator.h` for memory management

Added `third_party/gpu_iface/` with dispatch.cuh, macros.hpp, math_ops.hpp, utils.cuh, vec_dtypes.hpp

Compile flags: added `-DPLATFORM_HIP_DEVICE` and `-I third_party/gpu_iface`

### Other Fixes Applied
- `extraCudaBf16Data` -> `extraCudaData` in .hip files
- FlashInfer dispatch blocks in `fastllm-attention.hip` wrapped with `#ifdef USE_FLASHINFER` / `#endif`
- Removed `-DUSE_FLASHINFER` from compile flags (FlashInfer attention disabled)
- `build_scripts/compile_hip.py`: excluded `fastllm-linear-bf16-fast.hip` (included by other files)
- `fastllm-cuda.cuh`: `#ifdef __CUDACC__` block refactored to include USE_ROCM branch
