## 03:35 - Post-TopK Crash Debug

### Root Cause Found: `utf8_to_gbk` + Invalid UTF-8 → Stack Corruption

**Timeline:**
1. Added debug prints to `gemma4.cpp` after TopK and ToDevice(CPU), at ForwardBatch return, and in `basellm.cpp` ResponseBatch around ForwardBatch call
2. **4 ForwardBatch calls all succeed** (1 prefill + 3 decode for `-l 4`)
3. ResponseBatch decode loop exits normally: `reached output_token_limit=4, breaking`
4. `benchmark: ResponseBatch returned, outputs.size=1` — normal return
5. **Crash was in `utf8_to_gbk()`** — model output contains invalid/garbled UTF-8 tokens
6. `std::wstring_convert<std::codecvt_utf8>` fails on garbled UTF-8, corrupting stack
7. `STATUS_STACK_BUFFER_OVERRUN` (0xC0000409) triggered on function return (stack cookie check)

**Fix:** Skip `utf8_to_gbk`, use direct `printf` with UTF-8 output
