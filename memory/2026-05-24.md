### Missing Architecture Features in fastllm gemma4.cpp:

**1. Per-Layer Input (CRITICAL)**
- Gemma 4 has `embed_tokens_per_layer.weight` shape `[262144, 8960]`
  = `vocab_size × (num_layers × hidden_size_per_layer_input)` = 262144 × (35 × 256)
- Each layer has `per_layer_input_gate` (Linear 1536→256) + `per_layer_projection` (Linear 256→1536)
- Forward: `residual += per_layer_projection(act_fn(per_layer_input_gate(x)) * per_layer_input[i])`
- Per-layer input comes from `Embedding(input_ids, embed_tokens_per_layer)`, reshaped to `[batch, seq, 35, 256]`
- **Completely missing from fastllm** — this is the #1 cause of wrong logits

**2. KV Shared Layers (IMPORTANT)**
- `num_kv_shared_layers = 20`: layers 15-34 share KV cache from earlier layers
- In HF: shared KV layers skip k_proj/v_proj computation, reuse stored KV
- fastllm processes all layers independently — wrong for last 20 layers
- Related: `attention_k_eq_v` — in full_attention layers with shared KV, k=v (only k_proj, no v_proj)

**3. Partial Rotary Factor**
- `full_attention` layers: `partial_rotary_factor = 0.25`
- Only 64 of 256 head dim pairs use RoPE (rest are 0 in sin/cos table)
- fastllm passes `rotaryDim = global_head_dim/2 = 256` — should be limited to partial
- Current fastllm code already creates the sin/cos with 0 padding, but LlamaRotatePosition2D uses `min(rotaryDim, m/2)` which may not correctly handle this
